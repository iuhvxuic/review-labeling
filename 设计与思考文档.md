# 一、数据集与任务描述
## 1.1 数据集选择
数据集名称：ChnSentiCorp（中文酒店评论情感分类数据集）   
数据来源：https://huggingface.co/datasets/dirtycomputer/ChnSentiCorp_htl_all   
数据规模：7,766 条中文酒店评论
## 1.2 输入特征
文本类型：中文酒店评论文本   
文本长度：平均 100-200 字   
内容特征：包含对酒店位置、房间、服务、价格、卫生等方面的主观评价
## 1.3 打标目标
任务类型：情感二分类   
标签定义：   
正面（positive）：表达满意、推荐、性价比高、愿意再次入住   
负面（negative）：表达不满、批评、投诉、不推荐
# 二、模型与方法
## 2.1 使用的基础模型
模型名称：通义千问 qwen-turbo   
选择理由：   
中文原生模型，对中文口语化表达理解准确   
性价比高，成本约 0.3元/百万tokens   
API稳定，响应速度快
## 2.2 调用方式  
调用类型：API 调用（通过 LangChain 框架）   
API 端点：阿里云 DashScope 兼容 OpenAI 接口   
参数设置：   
temperature: 0.1（低温度保证输出稳定）   
max_tokens: 50（只需输出一个数字）
## 2.3 Prompt 设计
系统提示词（System Prompt）：

你是一位情感分析专家。对中文酒店评论进行情感分类。   
分类标准：  
正面情感：输出 1  
负面情感：输出 0  
只输出一个数字，不要输出任何其他内容。

## 错误分析
典型错误模式：正面评论被误判为负面，例：   
```
1.早餐太差，无论去多少人，那边也不加食品的。酒店应该重视一下这个问题了。房间本身很好。  
2.总的来说，这样的酒店配这样的价格还算可以，希望他赶快装修，给我的客人留些好的印象  
3.酒店正在申定五星中，不过目前看来四星都有点勉强。大堂很气派，不过细节很粗糙。硬件需要加强。服务人员礼貌热情。酒店的房间和早餐都还不错。
```
错误规律总结：   
褒贬并存评论易和数据集标签出现矛盾：评论中既有正面也有负面内容时，模型倾向于输出负面，而数据集创作者倾向于输出正面。   
模型过于敏感：对任何批评性词语都过度反应。
## 改进方法
优化提示词，增加Few-shot示例，使模型打标时更符合数据集的评价方式。
